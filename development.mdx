## Dashboard (Next on Cloudflare)

```bash
cd cdp-as-a-service-dashboard
npm install
npm run dev
# Open http://localhost:3000
```

Environment used by the Dashboard server routes:

- <code>LISA_BASE</code> — base URL to Lisa (e.g., <code>https://lisa-taurine.tera.space</code>)
- <code>LISA_JWT_SECRET</code> — must match Lisa’s <code>CONSUMER_JWT_SECRET</code>

## Lisa (Cloudflare Worker)

```bash
cd Taurine/lisa
wrangler dev
# Or deploy: wrangler deploy
```

Required env bindings:

- <code>PROD_DB</code> (D1)
- <code>TRON_API_BASE</code>, <code>TRON_API_KEY</code>
- <code>BILLING_URL</code>, <code>BILLING_SERVICE_SECRET</code>
- <code>CONSUMER_JWT_SECRET</code>, <code>FUJI_API_KEY</code>

## Tron (Agents)

Use hosted or run locally:

```bash
cd Roadrunner-Tron
npm install
docker compose up -d redis
npm run docker:build:linux
docker compose up -d api worker
# API on http://localhost:3000
```

Lisa expects Tron endpoints:

- <code>POST /v1/task/create</code>
- <code>GET /v1/task/:id</code>

## Billing Worker

```bash
cd billing-worker
wrangler dev
# Or wrangler deploy
```

Set <code>SERVICE_SECRET_LISA</code>/<code>SERVICE_SECRET_DASHBOARD</code> and optional rate vars.

<Tip>
  For production, set <code>TRON_API_BASE=https://tron.tera.space</code> and use Bearer <code>123456</code> inside Lisa.
</Tip>
